# ฉันจะซื้อหวยทุกงวด — PWA วิเคราะห์สลากย้อนหลัง 30 ปี

เว็บแอปสำหรับมือถือโทนกวน ๆ “ระวังโดนหวยแดกนะคะ” ที่ใช้ข้อมูล 716 งวด (16 ก.ค. 2539–16 ก.ค. 2569) เพื่อจัดอันดับเลข 00–99 แบบโปร่งใส พร้อมตรวจผลงวดล่าสุดผ่าน API ของสำนักงานสลากกินแบ่งรัฐบาล


## ระบบเซียมซีเสี่ยงเลข (v1.4.3)

- กระบอกเซียมซีพร้อมแอนิเมชันเขย่า
- โหมดสุ่มล้วนด้วย `crypto.getRandomValues()`
- โหมดสุ่มผสมสถิติ ซึ่งยังคงเปิดโอกาสให้เลข 00–99 ทุกเลขถูกเลือกได้
- สุ่มเลข 2 ตัว, 3 ตัว, 6 หลัก และเลขประกอบ
- คำทำนายภาษาไทยแบบกวน ๆ พร้อมคำเตือนเรื่องงบประมาณ
- บันทึกประวัติ 10 ครั้งล่าสุดใน `localStorage`
- คัดลอกผล และนำเลข 2 ตัวไปเทียบกับอันดับโมเดลได้

## รันบนเครื่อง

```bash
node server.mjs
```

เปิด `http://localhost:4173`

ไม่ต้อง `npm install` เพราะใช้ Node.js built-in modules เท่านั้น

## ฟังก์ชัน

- หน้าแรกตาม mockup ธีมสาวอนิเมะชุดไทยและชนบทไทย
- จัดอันดับเลขท้าย 2 ตัว และเลขท้าย 2 ตัวของรางวัลที่ 1
- เลือกวันที่งวดและรูปแบบการให้น้ำหนักของโมเดล
- แสดงผลทดสอบย้อนหลังแบบเดินหน้า (walk-forward) และชุดกันไว้ทดสอบ (holdout)
- ตรวจผลล่าสุดผ่าน GLO API พร้อมระบบสำรองด้วยฐานข้อมูลออฟไลน์
- สถิติความถี่ หลักสิบ/หลักหน่วย และบริบทวัน–เดือน–ราศี
- ติดตั้งเป็น PWA บน Android/iOS/desktop

## Deploy

### Vercel (แนะนำสำหรับตรวจผลสด)
อัปโหลดโฟลเดอร์นี้ไป GitHub แล้ว Import Project ใน Vercel ได้ทันที ไฟล์ `api/latest.js`, `api/check.js` และ `api/result.js` ทำหน้าที่เป็นตัวกลาง (proxy) ไปยัง GLO API

โปรเจกต์มี `vercel.json` แบบขั้นต่ำสำหรับกำหนด URL และป้องกัน Service Worker/Manifest ค้างในแคช โดยไม่ใช้ rewrite แบบ source=destination ที่ไม่จำเป็น ทั้งนี้ Vercel ตรวจจับฟังก์ชันในโฟลเดอร์ `api/` ได้อัตโนมัติ

### GitHub Pages
ส่วนหน้าและข้อมูลย้อนหลังทำงานได้ แต่ ปลายทาง `/api/*` จะไม่มีระบบหลังบ้านแบบ serverless จึงใช้ตรวจแบบออฟไลน์เป็นหลัก

## หลักการโมเดล

คะแนนรูปแบบ (pattern score) ผสมจาก 4 ส่วน:

1. ความถี่ระยะยาวแบบปรับเรียบ (smoothing)
2. ความถี่ให้น้ำหนักงวดล่าสุด
3. วันในสัปดาห์ เดือน และราศี โดยปรับลดอิทธิพลของกลุ่มข้อมูลขนาดเล็ก (shrinkage)
4. ระยะห่างจากครั้งที่เลขนั้นออกล่าสุด โดยให้น้ำหนักต่ำ

คะแนนนี้ไม่ใช่ค่าความน่าจะเป็นจริง และไม่ได้รับประกันผลในอนาคต

## แหล่ง API ทางการ

- Latest draw: `POST https://www.glo.or.th/api/lottery/getLatestLottery`
- Check ticket: `POST https://www.glo.or.th/api/checking/getcheckLotteryResult`

## สิทธิ์การใช้ข้อมูล
ชุด API ของ GLO Data Catalog ระบุเงื่อนไข Creative Commons Non-Commercial (Any) จึงควรตรวจเงื่อนไขเพิ่มเติมก่อนนำแอปไปหารายได้หรือเผยแพร่เชิงพาณิชย์

## การแก้ไขสำคัญใน v1.4.3

- แก้ตัวแปลงผลล่าสุดของ GLO ให้เข้า `response.data` และอ่านเฉพาะ `number[].value` ไม่สแกนตัวเลขจาก JSON ทั้งก้อน
- รองรับวันที่ภาษาไทย เช่น `16 กรกฎาคม 2569`, ชื่อเดือนย่อ และเลขไทย
- ตรวจผลออนไลน์แบบมีโครงสร้าง และ fallback ไปยัง API ผลเต็มงวด เพื่อครอบคลุมรางวัลข้างเคียงรางวัลที่ 1 และรางวัลที่ 3–5
- โหมดออฟไลน์แสดงข้อจำกัดชัดเจนและเตือนไม่ให้ทิ้งสลาก
- การจัดอันดับงวดในอดีตใช้เฉพาะข้อมูลก่อนวันที่เลือก ป้องกัน look-ahead bias
- ผล backtest มีป้ายชัดเจนว่าเป็นของโหมดสมดุล
- Service Worker ไม่ cache HTTP error/404/206 และเซิร์ฟเวอร์ตรวจ path ด้วยขอบเขตโฟลเดอร์ที่ถูกต้อง
- เพิ่มคำแนะนำ Add to Home Screen สำหรับ Safari บน iPhone/iPad

npm test

## ระบบขูดต้นมะขามขอเลข

- ขูดผิวเปลือกจำลองบน Canvas ด้วยนิ้วหรือเมาส์
- ตัวเลขสร้างด้วย `crypto.getRandomValues()` ก่อนเริ่มขูด การขูดเป็นเพียงการเปิดเผยผล
- มีโหมดสุ่มล้วนและสุ่มผสมสถิติแบบอิทธิพลต่ำ
- เก็บประวัติในอุปกรณ์สูงสุด 10 รายการ พร้อมคัดลอกและเทียบอันดับสถิติ
- เป็นความบันเทิงและไม่สนับสนุนการทำร้ายต้นไม้จริง
